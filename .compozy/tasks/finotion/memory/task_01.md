# Task 01 — Xcode Project Scaffold

## Objective Snapshot

Create the Xcode project skeleton with all required capabilities, entitlements, Info.plist keys, and folder structure. No user-visible features — pure infrastructure.

## Important Decisions

- Used **XcodeGen** (`project.yml`) as the project generator instead of committing a binary `.xcodeproj`. The `.xcodeproj` is generated by running `xcodegen generate` inside `Finotion/` on macOS.
- `SWIFT_TREAT_WARNINGS_AS_ERRORS = YES` applied to both Debug and Release configurations via XcodeGen settings block.
- SwiftLint configured as a `preBuildScripts` entry in `project.yml`; `.swiftlint.yml` at `Finotion/` root.
- Background modes (`fetch`, `remote-notification`) are in `UIBackgroundModes` (Info.plist), not in the entitlements — confirmed standard Apple convention.
- `.gitkeep` files in `Features/`, `Services/`, `Core/`, `Models/` to preserve empty folder groups until content is added in subsequent tasks.

## Learnings

- Build + simulator verification cannot run on Windows — all source/config created; `xcodegen generate` + `xcodebuild` must run on macOS.
- XcodeGen excludes `.gitkeep` files from the compiled sources list via `excludes: ["**/.gitkeep"]`.

## Files / Surfaces

- `Finotion/project.yml` — XcodeGen project definition (run `xcodegen generate` here)
- `Finotion/.swiftlint.yml` — lint rules
- `Finotion/Finotion/FinotionApp.swift` — `@main` entry point
- `Finotion/Finotion/ContentView.swift` — placeholder root view (replaced in task_05/task_11)
- `Finotion/Finotion/Info.plist` — URL scheme, background task ID, background modes
- `Finotion/Finotion/Finotion.entitlements` — iCloud CloudKit, iCloud KV, Keychain Sharing
- `Finotion/Finotion/Features/`, `Services/`, `Core/`, `Models/` — folder groups (empty with .gitkeep)
- `Finotion/FinotionTests/FinotionTests.swift` — baseline XCTest target
